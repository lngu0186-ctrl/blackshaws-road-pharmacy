# Web UX audit

Use this file when reviewing usability, layout clarity, accessibility, and trust.

## Accessibility basics

- Every icon button needs an accessible name.
- Inputs need labels, hints where useful, and clear error text.
- Focus states must be visible.
- Colour contrast must survive mobile/outdoor viewing.
- Keyboard navigation should work for menus, modals, and forms.

## Mobile review

- Tap targets should be comfortably pressable.
- Avoid dense clusters of controls near each other.
- Respect safe spacing and readable line lengths.
- Check sticky elements do not cover content or CTAs.

## Forms and conversion

- Ask only for needed fields.
- Use suitable input types and autocomplete.
- Put reassurance near sensitive fields.
- Show success, loading, and error states clearly.
- Make the primary CTA obvious and consistent.

## Visual hierarchy

- One clear primary action per section.
- Headings should say something useful, not generic filler.
- Support copy should reduce doubt, not add noise.
- Repetition in card layouts should feel consistent.
- Empty states should still guide the user forward.

## Trust and professionalism

For pharmacy and healthcare-adjacent experiences especially, check:

- contact details are easy to find
- booking/refill/service steps are obvious
- reassurance copy is present where users may hesitate
- policy, privacy, and safety information is discoverable
- pages do not feel visually cluttered or "template-ish"

## What to report

- UX issue
- User impact
- Screen/component area
- Suggested wording/layout fix
- Priority
